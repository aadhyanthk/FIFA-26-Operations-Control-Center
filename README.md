# FIFA 26 Operations Control Center

The **FIFA 26 Operations Control Center** is a standalone Windows desktop application designed to simulate and manage operations for a ~60,000-seat stadium (modeled on MetLife Stadium) during the FIFA World Cup 2026.

## Features

1. **Stadium Simulator**: A causal, event-driven engine simulating weather, transport, arrivals, gates, crowds, medical, security, cleaning, and food subsystems.
2. **Operations Dashboard**: Real-time metrics, a 2D HTML Canvas stadium map, incident feed, and predictive alerts.
3. **AI Operations Agent**: An integrated LLM agent (powered by local Ollama) that monitors the simulation, proposes execution plans, and actions them after human approval.

## Tech Stack
- **Framework**: Tauri v2 (Rust shell) + React 18 + Vite
- **Language**: TypeScript
- **State Management**: Zustand
- **Styling**: Vanilla CSS (CSS custom properties, dark theme)
- **AI Integration**: Ollama (`phi3:mini` by default) via HTTP fetch

## Setup & Execution

### Prerequisites
1. **Node.js** (v18+)
2. **Rust & Cargo** (for Tauri backend)
3. **Ollama**: You must have Ollama installed and running locally to use the AI Agent features. 

### Option 1: Standalone Desktop App (Windows)
1. Install [Ollama](https://ollama.com/) and run: `ollama pull phi3:mini`
2. Ensure you have Node.js and Rust installed (for Tauri).
3. Run `npm install`
4. Run `npm run tauri build`
5. The installer will be located in `src-tauri/target/release/bundle/`.

### Option 2: Local Network Deployment (Docker)
For control rooms where multiple tablets/devices need access to the dashboard without installing Ollama on every device, you can deploy the app centrally via Docker:

1. Ensure Docker and Docker Compose are installed on the host machine.
2. Run `docker-compose up -d --build`
3. This will automatically:
   - Build the React dashboard and serve it via Nginx on port `3000`.
   - Spin up an Ollama container, expose port `11434`, and automatically download the `phi3:mini` model.
4. Access the dashboard from any device on the local network by navigating to `http://<HOST_IP>:3000`.

## Agent Workflows
The AI agent operates using an Observe-Orient-Decide-Act (OODA) loop. You can prompt the agent directly using the Command Bar at the top of the interface (e.g., `/summary`), or wait for it to proactively respond to critical incidents generated by the stadium simulation engine.
